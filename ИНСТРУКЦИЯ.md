# 🚢 МОРСКОЙ БОЙ — Инструкция по установке

## 📐 Архитектура

```
[Браузер игрока]  ←→  [HTML/JS frontend]  ←→  [Google Apps Script API]  ←→  [Google Sheets]
```

- **Google Sheets** — база данных: хранит игроков, состояние, журнал ходов
- **Apps Script** — бэкенд API: обрабатывает входы, ходы, античит-логику
- **HTML файл** — фронтенд: интерфейс, опрашивает API каждые 1.2 сек

---

## 📋 Шаг 1 — Создать Google Таблицу

1. Откройте [sheets.google.com](https://sheets.google.com)
2. Нажмите **"+"** — создать новую таблицу
3. Назовите её: `Морской Бой`
4. Скопируйте **ID таблицы** из URL:
   ```
   https://docs.google.com/spreadsheets/d/  ЭТОТ_ДЛИННЫЙ_ID  /edit
   ```
   *(ID нужен только если вы привязываете скрипт вручную)*

---

## 📋 Шаг 2 — Создать Apps Script

**Способ А (рекомендуется) — скрипт привязан к таблице:**

1. Откройте вашу таблицу `Морской Бой`
2. В меню: **Расширения → Apps Script**
3. Откроется редактор скриптов с файлом `Код.gs`
4. **Полностью удалите** весь существующий код
5. Вставьте содержимое файла `Code.gs` из этого пакета
6. Сохраните: `Ctrl+S` или кнопка **Сохранить проект** (иконка дискеты)

---

## 📋 Шаг 3 — Настроить пароль администратора

В начале файла `Code.gs` найдите строку:

```javascript
var ADMIN_PASSWORD = "admin123";
```

Замените `"admin123"` на свой пароль.

---

## 📋 Шаг 4 — Задеплоить как Web App

1. В редакторе Apps Script нажмите кнопку **"Развернуть"** (Deploy) → **"Новое развёртывание"**
2. Нажмите иконку шестерёнки ⚙ рядом с "Тип" → выберите **Веб-приложение**
3. Заполните настройки:
   - **Описание:** `Морской Бой v1`
   - **Выполнять как:** `Я (ваш аккаунт)`
   - **Кто имеет доступ:** `Все` *(обязательно!)*
4. Нажмите **Развернуть**
5. При запросе разрешений: нажмите **Разрешить доступ**
   - Если появится предупреждение "Небезопасное приложение" → нажмите "Дополнительно" → "Перейти к Морской Бой (небезопасно)"
6. Скопируйте и сохраните **URL веб-приложения**:
   ```
   https://script.google.com/macros/s/AKfyc.../exec
   ```

---

## 📋 Шаг 5 — Подключить фронтенд

1. Откройте файл `index.html` в любом текстовом редакторе
2. Найдите строку:
   ```javascript
   var API_URL = "ВСТАВЬТЕ_СЮДА_URL_ВАШЕГО_WEB_APP";
   ```
3. Замените `"ВСТАВЬТЕ_СЮДА_URL_ВАШЕГО_WEB_APP"` на скопированный URL:
   ```javascript
   var API_URL = "https://script.google.com/macros/s/АБВ.../exec";
   ```
4. Сохраните файл

---

## 📋 Шаг 6 — Запустить игру

### Вариант A — Открыть локально
Просто откройте `index.html` в браузере двойным кликом.

### Вариант B — Разместить онлайн (для игры с другом)
Загрузите `index.html` на любой хостинг:
- [GitHub Pages](https://pages.github.com) — бесплатно
- [Netlify Drop](https://app.netlify.com/drop) — перетащите файл
- Любой веб-сервер

---

## 🎮 Как тестировать вдвоём

1. Откройте `index.html` в **двух разных** браузерах (или вкладках в режиме инкогнито)
2. **Игрок 1:** введите никнейм → нажмите "Войти в игру"
3. **Игрок 2:** введите другой никнейм → нажмите "Войти в игру"
4. После входа второго игрока — игра начинается автоматически
5. Ходит тот, у кого есть надпись **(ВАШ ХОД)** в статус-баре
6. Кликните по клетке на поле противника для выстрела

---

## ⚡ Принудительный вход (если игра заполнена)

Если 2 игрока уже в игре, а вам нужно войти:
1. Введите свой никнейм
2. Введите пароль администратора
3. Выберите слот для кика (1 или 2)
4. Нажмите **"Принудительный вход"**

1. Введите пароль администратора

## 🗂 Структура Google Sheets (создаётся автоматически)

| Лист | Колонки | Назначение |
|------|---------|------------|

## ❗ Частые проблемы

**"Ошибка подключения. Проверьте URL API"**
→ URL в `index.html` неверный или не вставлен

**CORS-ошибка в консоли**
→ Убедитесь что "Кто имеет доступ" = **Все** (не "Только авторизованные")

**Игра не обновляется**
→ Проверьте консоль браузера (F12). Apps Script иногда работает медленно (~2-3 сек)

**После изменения кода скрипт не обновился**
→ После каждого изменения нужно создавать **новое развёртывание** ("Развернуть" → "Новое развёртывание")

**Таблица не заполняется**
→ Запустите функцию `initSheets()` вручную из редактора Apps Script один раз (кнопка ▶)

## 🔒 Античит (реализовано на бэкенде)

- Нельзя ходить не в свою очередь — сервер отклонит запрос
- Нельзя стрелять в одну клетку дважды — сервер проверяет историю выстрелов
- При попадании — ход остаётся у атакующего (BLOCK_NEXT)
- При промахе — ход передаётся противнику (ALLOW_NEXT)
- Принудительный вход проверяется только на бэкенде (пароль не хранится на фронте)
