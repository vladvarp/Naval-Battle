# NAVAL COMBAT — Морской бой

Веб-игра «Морской бой» на чистом HTML/CSS/JavaScript с бэкендом на **Google Apps Script** и хранением в **Google Таблицах**. Поддерживаются **онлайн-комнаты** (до двух игроков на комнату), **соло против компьютера** (без сервера) и обширная **звуковая подсистема** с кэшированием через Service Worker.

---

## Архитектура

```
[Браузер]  ←→  [index.html + scripts/** + styles/**]  ←→  [Web App Apps Script]  ←→  [Google Sheets]
```


| Слой                                | Роль                                                                      |
| ----------------------------------- | ------------------------------------------------------------------------- |
| **Google Sheets**                   | Листы `Комнаты`, `Игроки`, `Журнал`: комнаты, игроки, выстрелы, фазы      |
| **Apps Script** (`backend/Code.gs`) | REST-подобный API: комнаты, ходы, админ, выход                            |
| **Фронтенд**                        | Лобби, PvP, соло-ИИ, UI, аудио (v1 HTML Audio / v2 Web Audio), мониторинг |


Конфигурация API: в файле `scripts/client.js` заданы `API_URL`, интервал опроса состояния (`POLL_INTERVAL`, по умолчанию 800 мс) и обновление списка комнат в лобби (`LOBBY_INTERVAL`, 10 с).

## Быстрый старт (только фронтенд)

Для корректной работы Service Worker с аудио и относительных путей к `./audio/*.mp3` страницу нужно отдавать с **корня сайта** (не открывать `index.html` как `file://`).

**Локально (Python):**

```bash
python -m http.server 8080
```

или, например:

```bash
py -m http.server 8081 --bind 127.0.0.1
```

Затем откройте в браузере `http://localhost:8080` (или указанный порт).

Соло-режим («Играть с компьютером») работает без развёрнутого бэкенда. Онлайн-игра требует настроенный Web App и `API_URL` в `scripts/client.js`.

## Установка бэкенда (Google)

### 1. Создать таблицу

1. [sheets.google.com](https://sheets.google.com) → новая таблица, например «Морской бой».
2. ID таблицы в URL пригодится, если скрипт привязываете вручную:
  `https://docs.google.com/spreadsheets/d/ЭТОТ_ID/edit`

### 2. Подключить Apps Script

**Рекомендуется — скрипт привязан к таблице:**

1. В таблице: **Расширения → Apps Script**.
2. Удалите код по умолчанию и вставьте содержимое файла `**backend/Code.gs`** из репозитория.
3. Сохраните проект (`Ctrl+S`).

### 3. Пароль администратора

В начале `Code.gs`:

```javascript
var ADMIN_PASSWORD = "admin";
```

Замените на свой надёжный пароль. Он используется для **списка комнат** и **удаления комнаты** через API, а не хранится на клиенте в открытом виде как «секрет приложения» — это только серверная проверка.

### 4. Развернуть как веб-приложение

1. **Развернуть** → **Новое развёртывание** → тип **Веб-приложение**.
2. **Выполнять как:** ваш аккаунт. **Кто имеет доступ:** **Все** (иначе браузер получит CORS/доступ ошибки).
3. Скопируйте URL вида `https://script.google.com/macros/s/.../exec`.

### 5. Подключить URL к клиенту

В `**scripts/client.js`** найдите:

```javascript
var API_URL = "...";
```

Вставьте URL вашего развёртывания. После **каждого** изменения кода в Apps Script создавайте **новое развёртывание** (или обновляйте существующее по правилам Google), иначе по URL будет отдаваться прежняя версия скрипта.

### 6. Первичная инициализация таблицы

Если листы не создались автоматически при первых запросах, в редакторе Apps Script один раз выполните функцию `**initSheets()`** (кнопка «Выполнить»).

## Как играть

### Онлайн (два игрока)

1. Оба игрока заходят на один и тот же хост с игрой.
2. Ввод никнейма → лобби.
3. Игрок A: **«Создать онлайн-комнату»** — получает **код комнаты** (5 символов).
4. Игрок B: в списке **«Комнаты»** нажимает **«Войти»** у нужной строки (код комнаты виден в списке и у создателя на экране ожидания — им можно поделиться голосом/чатом).
5. После подключения второго игрока фаза `playing`: первым ходит создатель комнаты (слот 1). Клик по полю противника — выстрел.
6. Комнаты в ожидании второго игрока со временем очищаются при **неактивности ~10 минут** (см. `ROOM_TIMEOUT_MS` в `Code.gs`).

### Соло с компьютером

В лобби кнопка **«Играть с компьютером»** — логика и сохранение партии локально (`localStorage`), без обращений к Sheets.

### Дополнительно в интерфейсе (без отдельной настройки)

Анимация снаряда (`#projectileCanvas`), шкала **вероятности победы**, полноэкранный режим, сворачиваемый журнал ходов, восстановление сессии онлайн-игрока из `localStorage` после перезагрузки страницы (при неизменённом API и живой комнате).

## Администрирование

В игре раскройте **«Администрирование»**:

- **Активные комнаты** — после ввода пароля загружается таблица комнат; кнопка **«Удалить»** вызывает действие `restart` с `roomId` (комната и игроки удаляются на сервере).
- **Монитор** / **Трекер** — диагностика клиента и сетевых запросов.
- **GAS Ping Monitor** — индикатор задержки до Google Apps Script в PvP (с порогами качества и предупреждением при длительной/критической задержке); при включённом `NET` автоматически стыкуется справа от него.
- **NET (Network Tracker Quick Button)** — отдельный плавающий виджет открытия `NETWORK TRACKER`, доступен на всех экранах; по умолчанию выключен и включается/выключается кнопкой `NET` в шапке трекера (сохранение через `localStorage`).
- **Аудио-кэш** / **Рандом** — состояние загрузки звуков и веса случайного выбора.
- Переключатель движка звука (**HTML Audio** ↔ **Web Audio**) — перезагрузка страницы; версия сохраняется в `localStorage` (`navalAudioEngine`).

Принудительного «кика» слота с ником и паролем в текущей версии UI **нет** — используйте удаление комнаты админом или дождитесь таймаута/выхода игроков.

## Структура Google Sheets (создаётся через `initSheets` / первые запросы)


| Лист        | Назначение                                                                                             |
| ----------- | ------------------------------------------------------------------------------------------------------ |
| **Комнаты** | `roomId`, игроки 1–2, фаза, активность, JSON выстрелов, победитель, очередь хода (`turn` — `playerId`) |
| **Игроки**  | `playerId`, ник, `roomId`, слот, JSON поля кораблей, `lastSeen`                                        |

Состояние партии для онлайна хранится в строке комнаты и у игроков; отдельный лист «Состояние» в текущем `initSheets` **не создаётся**. В `Code.gs` объявлена неиспользуемая константа `SHEET_NAME_STATE` — лист под неё скрипт сам не создаёт.

## Расстановка кораблей (`placement_setup.js`)

Перед каждой партией (и соло, и онлайн) открывается оверлей расстановки через `window.openPlacementSetup(opts)`. Функция возвращает `Promise`, который разрешается двумерным массивом `10×10` (1 — клетка корабля) или `null` (отмена).

**Режим по умолчанию — ручной.** Все корабли стартуют в лотке внизу; игрок расставляет их сам.

### Управление

| Действие | Desktop | Mobile |
|---|---|---|
| Переместить корабль | Перетащить мышью | Перетащить пальцем |
| Повернуть корабль на поле | Двойной клик | Двойное касание (< 400 мс) |
| Случайная расстановка | Кнопка **🎲 РАНДОМ** | Кнопка **🎲 РАНДОМ** |

- Перетаскивание из лотка: начинается немедленно при нажатии на корабль в лотке.
- Перетаскивание с поля: начинается после смещения указателя более чем на 6 px (чтобы не конфликтовать с двойным кликом/тапом).
- Подсветка предпросмотра: зелёная — место свободно, красная — столкновение или выход за границы.
- Корабли с конфликтом подсвечиваются красным; кнопка **ГОТОВО** блокируется до устранения всех столкновений.
- Кнопка **ГОТОВО** также показывает подсказки: «РАССТАВЬТЕ ВСЕ КОРАБЛИ» / «ИСПРАВЬТЕ СТОЛКНОВЕНИЯ».

### Touch-реализация (мобильные)

Помимо стандартных `pointerdown/move/up`, модуль отдельно слушает `touchstart/touchmove/touchend` с `{ passive: false }`, что позволяет вызывать `e.preventDefault()` и блокировать скролл страницы во время перетаскивания. Двойной тап реализован через таймер (`lastTap`, порог 400 мс) внутри обработчика `touchstart`.

## Поведение сервера (кратко)

- Ход только у игрока, чей `playerId` совпадает с полем `turn` в комнате.
- Повторный выстрел в ту же клетку отклоняется.
- При **попадании** ход остаётся у стрелявшего; при **промахе** передаётся противнику.
- При потоплении корабля результат `sunk`, на сервере обновляется доска противника; клиент отображает периметр вокруг потопленного (логика согласована с бэкендом).

## Частые проблемы


| Симптом                                   | Что проверить                                                        |
| ----------------------------------------- | -------------------------------------------------------------------- |
| Ошибка подключения к API                  | `API_URL` в `scripts/client.js`, доступность Web App                 |
| CORS / блокировка                         | У развёртывания доступ **«Все»**, не только аккаунт Google           |
| Медленные обновления                      | Задержки Apps Script (1–3 с возможны); интервал опроса в `client.js` |
| Длительная задержка в индикаторе GAS      | Проверить сеть/канал, затем перезагрузить страницу                    |
| Звуки не с кэшем / не с SW                | Открывайте сайт через `http(s)://`, не `file://`                     |
| На iPhone нет звука                       | Режим «Беззвучный» и политика iOS к веб-аудио — см. подсказку в игре |
| После правки `Code.gs` ничего не меняется | Новое развёртывание или обновление версии Web App в Google           |
| После правки фронтенда не видно изменений  | Hard reload (`Ctrl+F5`) или версионировать script URL через `?v=...` |
| Комната удалена админом, игрок "застрял"   | Клиент теперь при `state: { ok:false, error:"Комната не найдена" }` автоматически выходит в лобби |
